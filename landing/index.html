What it is

Research swarm by default.
Auto-trader by deliberate choice.

01

15 specialized agents

System check, protocol research, subnet discovery and scoring, watch-only wallet, market analysis, risk review, miner / validator engineering, training, infra, dashboard, fullstack, QA, docs.

02

Gate before route

Every action is classified SAFE / CAUTION / DANGER before the router ever sees it. In read-only modes DANGER outputs a plan only. In opt-in AUTO_TRADING mode DANGER routes to the audited execution path only when kill-switch off, position cap set, and daily-loss-limit set.

03

Plug-in architecture

Drop your own *_agent.py in a folder, set TAO_PLUGIN_PATHS, done. Or ship as a pip package via the tao.agents entry point. Plug-ins go through the same gate as built-ins. No privilege escalation.

04

Four wallet modes, opt-in

Default is NO_WALLET. WATCH_ONLY takes public SS58 only. MANUAL_SIGNING emits unsigned plans for you to sign externally. AUTO_TRADING uses a dedicated encrypted keystore (Argon2id + AES-256-GCM) and signs only when env var, keystore unlock, and per-strategy opt-in are all set. Seed phrases and main coldkey keys are never asked for, stored, or transmitted.

05

Live data, real fallbacks

Bittensor SDK 10.x against mainnet finney for 129+ subnets with rich identity. Subscan for wallet snapshots. CoinGecko for prices. Each path falls back to mock fixtures with a labelled _meta.fallback_reason when the upstream is unreachable — never silent.

06

Local-only, no telemetry

Everything runs on your machine. SQLite for state. No analytics. No vendor SDKs that phone home. Outbound HTTPS only to the data endpoints you explicitly opt into via --live.

Three commands.

python -m venv .venv && source .venv/bin/activate

    --tick-interval-s 60 \
    --status-file data/runner_status.json
# in another terminal:

Trading pipeline

Paper by default.
Live behind three independent opt-ins.

Sixteen merged PRs (2A–2P) build the audited execution path on top of the read-only swarm. Nothing signs anything until every gate consents.

01

Encrypted keystore

Argon2id (OWASP 2024 baseline: t=3, m=64 MiB, p=4) plus AES-256-GCM. Atomic write, 0o600 permissions, anti-timing failure mode. Same-exception for wrong password and tampered ciphertext.

02

Three-stage live gate

TAO_LIVE_TRADING=1 in env, --keystore-path with the unlock password, StrategyMeta.live_trading=True. Plus a typed I UNDERSTAND confirmation in the CLI before the first extrinsic.

03

Cold-start reconciliation

On the first tick after every restart the runner reads the actual on-chain stake from get_stake_info_for_coldkey and rewrites its position book. No more "started fresh, overshot the cap by my entire current stake".

04

Slippage + chain verification

rate_tolerance + allow_partial on every proposal. After every successful broadcast, an optional re-read confirms the on-chain delta matches direction within tolerance — mismatches land as _verification_failed audit rows.

05

Adaptive ensemble

Run multiple strategies in parallel; weights come from recent realised P&L via inverse_loss_weights. Winners get more capital, losers throttle (with a floor for regime change). Per-base attribution in the audit ledger.

06

Dashboard with halt

Streamlit UI with always-visible runner-status pill, equity curves, per-strategy KPIs, suggested-weight bars, an in-page backtest harness, and a one-click halt button that touches the kill-switch file.

Roadmap & monetization

Honest plan. No tokens. No managed funds.

We do not have a token and will not launch one. We do not manage anyone's funds. Auto-trading is a single-user feature running on your own machine with your own hot key. Long-term sustainability comes from commercial licensing for hosting / multi-user / managed-fund use cases — not from speculation.
